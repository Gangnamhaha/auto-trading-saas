# 한국 자동매매 SaaS 법률/규제 검토 보고서

## 1. 요약 (Executive Summary)

본 서비스(사용자 전략 설정 -> 플랫폼이 KIS API로 자동 주문 실행)는 구현 방식에 따라 자본시장법상 **단순 소프트웨어 제공(도구)**, **투자자문업**, **투자일임업**으로 법적 성격이 달라진다. 핵심 리스크는 (a) 개별 투자판단 제공 여부, (b) 주문 실행권한의 실질 귀속, (c) 수수료 구조/광고 문구다.

1인 창업 및 초기 자본 1,000만원 미만 조건에서는 즉시 투자일임업 인허가 트랙으로 가기 어렵다. 초기에는 **비개별화 도구형(Execution Tool) + 강한 면책/광고 통제 + 주문권한 비보유 구조**가 현실적이며, 투자조언/자동일임으로 해석될 수 있는 기능은 단계적으로 분리해야 한다.

## 2. 규제 분류 분석 (자본시장법 기준)

아래 조문은 본 서비스 분류에 직접적인 기준이 된다.

- **자본시장법 제6조제7항**: 투자자문업(금융투자상품 가치/투자판단 자문에 응하는 영업)
- **자본시장법 제6조제8항**: 투자일임업(투자판단의 전부/일부를 일임받아 운용)
- **자본시장법 제18조**: 금융투자업(투자자문업/투자일임업 포함) 등록 관련 기본 조문
- **자본시장법 제98조**: 투자일임업자 행위규율 관련 핵심 조문군(일임 운용 관련 준수의무)
- **자본시장법 제101조제1항**: 유사투자자문업(불특정다수 대상 조언) 정의
- **자본시장법 제444조**: 무인가/무등록 영업 관련 형사처벌 조문(중대 리스크)

구분 기준 요약:

- **거래 도구(합법 가능성 높음)**: 사용자 스스로 규칙을 입력하고, 플랫폼은 기술적 자동집행만 수행. 개별 종목 추천/상담 없음.
- **투자자문업 해당 가능**: 사용자 질문/성향에 맞춘 개별 종목·매수시점·비중을 제시.
- **투자일임업 해당 가능**: 플랫폼/알고리즘이 사실상 투자판단을 대신하고 계좌 주문을 상시 자동 실행.

참고: 금융위/금감원 법령해석(유사투자자문업자의 자동매매 연동 불가) 및 대법원 판례(개별성 판단) 취지상, "개별성 + 실행권한"이 규제 강도를 결정한다.

## 3. 자본금 요건 및 인허가 절차

### 3.1 투자자문업 등록 요건(실무)

- 요건은 법 본문 + 시행령 + 감독규정(업무단위/대상투자자별)으로 확정된다.
- 실무상 검토 항목: 최소 자기자본, 전담 인력(컴플라이언스/리스크), 전산설비, 내부통제기준, 이해상충 방지체계.
- 시장에서 통용되는 가이드로는 투자자문업/투자일임업 모두 **자기자본 요건이 상당**하며, 특히 일반투자자 대상 투자일임은 초기 스타트업에 높은 진입장벽이다.

### 3.2 FSC 등록 절차(개요)

1. 사업모델 확정(자문 vs 일임 vs 유사자문 범위)
2. 법무법인/컴플라이언스 자문 통해 사전갭 분석
3. 내부통제/약관/위험고지/광고심의 프로세스 구축
4. 금융위 등록 신청(필요시 금감원 사전협의)
5. 보완요구 대응 후 등록

결론: 현재 자본/인력 조건에서는 정식 투자일임업 진입보다, 규제저촉을 회피한 도구형 SaaS가 실행가능성이 높다.

## 4. 서비스 구조별 법적 리스크 평가

### 구조 1: 순수 도구 (사용자 API 키 사용) -> LOW

- 조건: API 키는 사용자 명의, 주문 트리거 규칙도 사용자 직접 설정, 플랫폼은 기계적 전송만 수행.
- 금지: 개별 종목 추천, 목표수익 보장, 1:1 투자판단 코칭.
- 필요 통제: 추천성 UI 제거, 백테스트 성과표시 가이드, 로그 보관.

### 구조 2: 신호 제공(랭킹/매수·매도 알림) -> MEDIUM

- 불특정다수 대상 일반 정보 제공은 가능 여지 있으나,
- 개인화된 "지금 이 종목 매수" 형태로 개별화되면 투자자문업 해당 위험이 상승.
- 신호-자동주문 직결 기능은 투자일임/무등록 영업 해석 리스크가 커짐.

### 구조 3: 역외 법인 + 국내 사용자 대상 제공 -> MEDIUM

- 서버/법인이 해외라도 한국 투자자 대상 영업이면 국내 규제 적용 리스크 존재.
- 결제/마케팅/고객지원이 국내 중심이면 실질 국내영업으로 판단될 수 있음.
- 조세/외환/전자금융/개인정보 국외이전 이슈까지 복합 발생.

## 5. 국내 유사 서비스 사례 분석

- **이루다투자(업라이즈 계열)**: 투자일임/자문 라이선스 기반 서비스 구조를 채택한 사례로 알려져 있음. 계약 체결, 위험고지, 투자성향확인 절차를 전면 배치.
- **불리오(두물머리투자자문)**: 투자자문업 등록 기반으로 앱 서비스 운영 및 자문계약 연계.
- **파운트**: 로보어드바이저/자문·일임 구조를 제도권 틀 안에서 운영한 대표 사례군.

시사점: 국내에서 "자동화 + 개인투자자 대상" 모델은 대체로 등록업(자문/일임) 체계 또는 강한 비자문형 도구 모델로 양분된다.

## 6. 권장 법적 구조 및 면책 전략

권장안(초기): **비자문형 거래자동화 도구 SaaS**

- 사용자 직접 입력한 규칙을 API로 실행하는 기술도구로 한정
- 개별 투자조언, 포트폴리오 추천, 수익률 보장 표현 금지
- 유료모델은 "소프트웨어 사용료" 중심(성과보수형 지양)
- 모든 투자판단 주체를 사용자로 명시
- 주문/신호 기능 분리(신호 -> 자동실행 기본 OFF)

확장안(중장기):

- 자본/인력 확보 후 투자자문업 등록 검토
- 고객 위임운용까지 할 경우 투자일임업 라이선스 별도 검토

## 7. 필수 면책 고지 문구 (최소 5개)

1. 이 서비스는 투자 원금 손실이 발생할 수 있습니다.
2. 과거 수익률이 미래 수익률을 보장하지 않습니다.
3. 본 서비스는 특정 금융투자상품의 매수·매도를 권유하거나 보장하지 않습니다.
4. 모든 투자판단과 최종 주문 책임은 이용자 본인에게 귀속됩니다.
5. 시장 급변, 통신 장애, 증권사/거래소 시스템 문제로 주문이 지연·실패할 수 있습니다.
6. 알고리즘/백테스트 결과는 실제 체결 환경(슬리피지, 수수료, 유동성)과 차이가 날 수 있습니다.
7. 회사는 법령상 허용 범위를 초과하는 손실보전·이익보장을 제공하지 않습니다.

## 8. 이용약관(ToS) 핵심 조항 초안 (최소 10개)

1. **서비스 이용 목적 제한**: 본 서비스는 기술적 자동주문 보조 도구로 제공되며 투자자문/일임을 제공하지 않는다.
2. **투자 결과 책임 귀속**: 투자판단, 주문설정, 전략 선택 책임은 전적으로 이용자에게 있다.
3. **서비스 중단 면책**: 시스템 점검/장애/외부 API 이슈 시 서비스가 중단될 수 있으며 회사의 고의·중과실 없는 손해는 면책된다.
4. **API 키 보안 책임**: API 키 발급/보관/폐기는 이용자 책임이며 유출로 인한 손해는 이용자 부담이다.
5. **개인정보 처리 동의**: 서비스 제공에 필요한 최소 정보 수집·이용 및 보관기간을 명시한다.
6. **분쟁 해결 방법**: 고객센터 이의제기 -> 협의 -> 조정/소송 절차 순으로 처리한다.
7. **준거법 및 관할 법원**: 대한민국 법령을 준거법으로 하며 관할 법원을 특정한다.
8. **서비스 변경/종료 권리**: 법령변경·사업상 필요 시 기능 변경, 중단, 종료 가능함을 고지한다.
9. **지적재산권**: 소프트웨어/콘텐츠/모델의 저작권 및 사용범위를 규정한다.
10. **이용 제한 사유**: 약관위반, 비정상 트래픽, 불법행위 의심 시 이용을 제한할 수 있다.
11. **광고 및 성과표시 제한**: 오인 가능 표현(원금보장, 확정수익 등)을 금지하고 과장광고를 제한한다.
12. **면책 범위의 한계**: 소비자보호법상 강행규정에 반하는 면책은 적용하지 않는다.

## 9. 개인정보처리방침 핵심 조항

PIPA(개인정보보호법) 기준으로 최소한 다음을 포함해야 한다.

- 수집 항목/목적/보유기간(최소수집 원칙)
- 제3자 제공 및 처리위탁 내역
- 국외 이전(클라우드/분석도구 포함) 시 이전 국가/항목/시점/방법
- 파기절차 및 방법
- 정보주체 권리(열람·정정·삭제·처리정지) 및 행사 방법
- 기술적·관리적 보호조치(접근통제, 암호화, 로그관리)

## 10. GO/NO-GO 권고

### 최종 권고: **조건부 GO**

GO 조건(모두 충족 필요):

1. 초기 출시 범위를 "비개별화 도구형 자동집행"으로 제한
2. 1:1 종목추천/개인화 투자조언/성과보수형 과금 기능 제외
3. 약관·면책·광고심의 문구를 론칭 전 법무 검토 완료
4. 주문실패/장애 대응 프로세스 및 로그보관 체계 구축
5. 라이선스 경계선(자문/일임) 침범 여부를 분기별 컴플라이언스 점검

NO-GO 트리거(즉시 중단/재설계):

- 플랫폼이 이용자 동의 없이 전략·종목·비중을 실질 결정하는 경우
- "수익 보장" 또는 손실보전 유사 문구를 마케팅에 사용하는 경우
- 유사투자자문업 범위를 넘어 개별상담 + 자동매매 연동을 결합하는 경우
