================================================================================
KEY IMPLEMENTATION DECISIONS
================================================================================

1. Next.js Config Format: .mjs (not .ts or .js)
   - Reason: package.json has "type": "module" (ES modules)
   - CommonJS (module.exports) requires .cjs extension

2. Web tsconfig.json: Explicit rootDir and vitest/globals types
   - Reason: Prevents TypeScript errors for test globals (describe, it, expect)
   - Includes: src/**/*.ts, src/**/*.tsx only

3. Turbo Pipeline: Separate tasks for build/test/lint/typecheck
   - Reason: Allows parallel execution and caching
   - build depends on ^build (dependencies first)
   - test/lint/typecheck have no dependencies (run in parallel)

4. Husky Hooks: Pre-commit + commit-msg
   - Reason: Enforce code quality (lint-staged) + commit message format (commitlint)
   - Prevents bad commits from entering history

5. Docker Compose: Alpine images (minimal, fast)
   - Reason: Smaller image size, faster startup
   - Health checks ensure services are ready before tests
