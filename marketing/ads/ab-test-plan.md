# AutoTrade KR — A/B 테스트 계획

> 최종 업데이트: 2026-03-15  
> 목표: 데이터 기반으로 전환율 최적화, 각 테스트 최소 통계적 유의성 확보

---

## A/B 테스트 원칙

1. **한 번에 하나의 변수만 테스트** — 여러 요소를 동시에 바꾸면 원인 파악 불가
2. **충분한 샘플 확보** — 각 변형 최소 100 방문자 (통계적 유의성 95%)
3. **테스트 기간 준수** — 최소 2주, 요일 효과 제거
4. **승자 즉시 적용** — 테스트 종료 후 3일 내 결과 반영
5. **결과 문서화** — 모든 테스트 결과를 이 문서에 기록

---

## 테스트 1: 랜딩페이지 헤드라인

### 개요

| 항목             | 내용                               |
| ---------------- | ---------------------------------- |
| 테스트 목표      | 무료 가입 전환율 최대화            |
| 측정 지표        | 가입 전환율 (방문자 → 이메일 등록) |
| 테스트 기간      | 2주                                |
| 최소 샘플        | 각 변형 100 방문자 (총 300 방문자) |
| 트래픽 배분      | 33% / 33% / 34%                    |
| 통계 유의성 목표 | 95%                                |

### 변형 설계

**변형 A (현재/기준)**

```
헤드라인: "한국 주식 자동매매, 이제 쉽게"
서브헤드: "코딩 없이 나만의 자동매매 전략을 만들고 백테스팅으로 검증하세요"
CTA: "무료로 시작하기"
```

_포지셔닝: 접근성 강조 (쉬움)_

**변형 B**

```
헤드라인: "코딩 없이 퀀트 투자 시작하기"
서브헤드: "전략 설정 → 백테스팅 → 자동 실행. 30일 무료 체험."
CTA: "무료로 시작하기"
```

_포지셔닝: 기술적 접근성 강조_

**변형 C**

```
헤드라인: "백테스팅으로 검증된 전략으로 자동매매"
서브헤드: "실제 투자 전 과거 데이터로 전략을 검증하세요. 무료 가입 후 즉시 시작."
CTA: "무료로 시작하기"
```

_포지셔닝: 신뢰/검증 강조_

### 가설

- **A**: 일반 투자자에게 친숙한 언어 → 넓은 타겟에 효과적
- **B**: 퀀트/기술 투자 관심자에게 어필 → 고의도 사용자 전환율 높음
- **C**: 리스크 회피 성향 투자자에게 어필 → 신중한 사용자 전환율 높음

### 측정 방법

```
도구: Google Optimize 또는 VWO
추적: GA4 이벤트 (sign_up_complete)
세그먼트: 신규 방문자만 (재방문자 제외)
```

### 결과 기록 템플릿

| 변형     | 방문자 | 가입 수 | 전환율 | 통계 유의성 |
| -------- | ------ | ------- | ------ | ----------- |
| A        | -      | -       | -      | -           |
| B        | -      | -       | -      | -           |
| C        | -      | -       | -      | -           |
| **승자** |        |         |        |             |

### 후속 액션

- 승자 헤드라인 → 모든 랜딩페이지에 적용
- 패자 변형 → 다음 테스트의 기준점으로 활용
- 세그먼트별 분석: 채널별(Google/Meta/YouTube/네이버) 승자가 다를 수 있음

## 테스트 2: CTA 버튼 문구

### 개요

| 항목             | 내용                                 |
| ---------------- | ------------------------------------ |
| 테스트 목표      | CTA 클릭률 및 가입 완료율 최대화     |
| 측정 지표        | CTA 클릭률 + 가입 완료율 (이중 측정) |
| 테스트 기간      | 2주                                  |
| 최소 샘플        | 각 변형 100 방문자 (총 300 방문자)   |
| 트래픽 배분      | 33% / 33% / 34%                      |
| 통계 유의성 목표 | 95%                                  |

### 변형 설계

**변형 A (현재/기준)**

```
버튼 텍스트: "무료로 시작하기"
버튼 색상: 브랜드 주 색상
위치: 히어로 섹션 중앙
```

_포지셔닝: 무료 강조, 낮은 진입 장벽_

**변형 B**

```
버튼 텍스트: "백테스트 무료 체험"
버튼 색상: 브랜드 주 색상
위치: 히어로 섹션 중앙
```

_포지셔닝: 핵심 기능(백테스팅) 강조_

**변형 C**

```
버튼 텍스트: "30일 무료 Pro 체험"
버튼 색상: 브랜드 주 색상
위치: 히어로 섹션 중앙
```

_포지셔닝: 프리미엄 체험 강조, 기간 명시_

### 가설

- **A**: 가장 일반적인 문구 → 기준 전환율
- **B**: 구체적 기능 언급 → 기능 관심 사용자 클릭률 높음
- **C**: "Pro" + "30일" 명시 → 가치 인식 높아 완료율 높을 수 있음

### 측정 방법

```
도구: Google Optimize + GA4
추적 이벤트:
  1. cta_click (버튼 클릭)
  2. sign_up_complete (가입 완료)
분석: 클릭률과 완료율 모두 측정 (클릭 후 이탈 파악)
```

### 결과 기록 템플릿

| 변형     | 방문자 | CTA 클릭 | 클릭률 | 가입 완료 | 완료율 | 통계 유의성 |
| -------- | ------ | -------- | ------ | --------- | ------ | ----------- |
| A        | -      | -        | -      | -         | -      | -           |
| B        | -      | -        | -      | -         | -      | -           |
| C        | -      | -        | -      | -         | -      | -           |
| **승자** |        |          |        |           |        |             |

### 후속 액션

- 클릭률과 완료율이 다른 변형에서 높을 경우: 완료율 우선
- 승자 CTA → 모든 페이지 CTA에 적용
- 버튼 색상 테스트 (다음 테스트로 연계)

## 테스트 3: 가격 표시 방식

### 개요

| 항목             | 내용                                     |
| ---------------- | ---------------------------------------- |
| 테스트 목표      | 유료 전환율 최대화                       |
| 측정 지표        | 유료 플랜 선택률 (무료 가입 → 유료 전환) |
| 테스트 기간      | 4주 (유료 전환 사이클 고려)              |
| 최소 샘플        | 각 변형 200 무료 가입자 (총 600명)       |
| 트래픽 배분      | 33% / 33% / 34%                          |
| 통계 유의성 목표 | 95%                                      |

### 변형 설계

**변형 A: 월 단위 표시 (현재/기준)**

```
Basic: 월 9,900원
Pro: 월 29,900원

표시 방식:
┌─────────────────┐  ┌─────────────────┐
│     Basic       │  │      Pro        │
│   ₩9,900/월    │  │  ₩29,900/월    │
│  [시작하기]     │  │  [시작하기]     │
└─────────────────┘  └─────────────────┘
```

**변형 B: 일 단위 환산 표시**

```
Basic: 하루 330원 (월 9,900원)
Pro: 하루 996원 (월 29,900원)

표시 방식:
┌─────────────────┐  ┌─────────────────┐
│     Basic       │  │      Pro        │
│  하루 330원     │  │  하루 996원     │
│ (월 9,900원)   │  │ (월 29,900원)  │
│  [시작하기]     │  │  [시작하기]     │
└─────────────────┘  └─────────────────┘
```

_심리: 커피 한 잔 가격 비교 효과_

**변형 C: 연간 할인 강조**

```
Basic: 연 89,000원 (월 7,416원, 25% 절약)
Pro: 연 269,000원 (월 22,416원, 25% 절약)

표시 방식:
┌─────────────────┐  ┌─────────────────┐
│     Basic       │  │      Pro        │
│  ₩89,000/년    │  │ ₩269,000/년    │
│ 월 7,416원     │  │ 월 22,416원    │
│ 💰 25% 절약    │  │ 💰 25% 절약    │
│  [시작하기]     │  │  [시작하기]     │
└─────────────────┘  └─────────────────┘
```

_심리: 할인 강조, 연간 약정으로 LTV 증가_

### 가설

- **A**: 기준 전환율, 월 단위로 부담 낮게 인식
- **B**: 일 단위 환산으로 심리적 가격 장벽 낮춤 → 전환율 높을 수 있음
- **C**: 연간 할인 강조 → 전환율은 낮을 수 있으나 LTV 높음

### 측정 방법

```
도구: GA4 + 결제 시스템 연동
추적 이벤트:
  1. pricing_page_view (가격 페이지 조회)
  2. plan_select (플랜 선택 클릭)
  3. payment_complete (결제 완료)
분석: 전환율 + 선택 플랜 분포 + 연간 vs 월간 비율
```

### 결과 기록 템플릿

| 변형     | 노출 | 플랜 선택 | 전환율 | Basic 비율 | Pro 비율 | 연간 비율 | 통계 유의성 |
| -------- | ---- | --------- | ------ | ---------- | -------- | --------- | ----------- |
| A        | -    | -         | -      | -          | -        | -         | -           |
| B        | -    | -         | -      | -          | -        | -         | -           |
| C        | -    | -         | -      | -          | -        | -         | -           |
| **승자** |      |           |        |            |          |           |             |

### 후속 액션

- 변형 C 승리 시: 연간 플랜 기본 표시로 변경
- 변형 B 승리 시: 일 단위 환산 + 월 단위 병기
- LTV 분석: 연간 플랜 선택자의 실제 갱신율 추적 (6개월 후)

## 테스트 관리 프로세스

### 테스트 우선순위 결정 기준

1. **예상 임팩트**: 전환율 개선 시 매출 영향도
2. **구현 난이도**: 개발 리소스 필요 여부
3. **테스트 기간**: 결과 확인까지 소요 시간
4. **데이터 신뢰도**: 충분한 샘플 확보 가능성

### 동시 진행 제한

- 같은 페이지에서 2개 이상 테스트 동시 진행 금지
- 서로 다른 페이지/채널은 동시 진행 가능

### 결과 분석 기준

```
통계적 유의성: 95% 이상 (p-value < 0.05)
최소 효과 크기: 전환율 10% 이상 개선
최소 샘플: 각 변형 100명 이상
```

### 테스트 결과 기록 위치

- 이 문서에 결과 섹션 추가
- GA4 대시보드에 테스트 기간 주석 추가
- 월간 마케팅 보고서에 요약 포함
