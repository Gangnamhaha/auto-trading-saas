# AutoTrade KR — ROI 추적 대시보드 설계

> 최종 업데이트: 2026-03-15  
> 도구: Google Looker Studio (주) + Mixpanel (보조)  
> 업데이트 주기: 실시간 (일부 지표 D+1)

---

## 대시보드 개요

광고 ROI를 실시간으로 추적하고 의사결정을 지원하는 통합 대시보드. 채널별 성과 비교, 전환 퍼널 분석, LTV 코호트 분석을 통해 예산 최적화를 지원한다.

### 대시보드 구성 (5개 섹션)

1. **Executive Summary** — 핵심 KPI 한눈에 보기
2. **채널별 CAC 비교** — 채널 효율 비교
3. **월별 ROAS 추이** — 광고 수익성 트렌드
4. **전환 퍼널 분석** — 드롭오프 포인트 파악
5. **코호트별 LTV 분석** — 장기 수익성 추적

## 섹션 4: 전환 퍼널 분석

### 차트 7: 전체 전환 퍼널

```
전환 퍼널 (이번 달)

광고 클릭/조회
    10,000명
       │ 30% 랜딩페이지 도달
       ▼
  랜딩페이지 방문
    3,000명
       │ 8% 가입 전환
       ▼
   무료 가입 완료
     240명
       │ 40% 브로커 연결
       ▼
   브로커 연결 완료
      96명
       │ 15% 유료 전환
       ▼
   유료 플랜 전환
      36명
```

**퍼널 단계별 지표**

| 단계                    | 사용자 수 | 전환율 | 드롭오프율 | 목표 전환율 |
| ----------------------- | --------- | ------ | ---------- | ----------- |
| 광고 클릭 → 랜딩 도달   | -         | -      | -          | 30%+        |
| 랜딩 → 무료 가입        | -         | -      | -          | 5%+         |
| 가입 → 브로커 연결      | -         | -      | -          | 40%+        |
| 브로커 연결 → 유료 전환 | -         | -      | -          | 15%+        |
| **전체 (클릭 → 유료)**  | -         | -      | -          | **0.3%+**   |

### 차트 8: 드롭오프 분석 (Sankey 다이어그램)

```
[광고 클릭] ──70%──→ [이탈]
           └─30%──→ [랜딩 방문] ──70%──→ [이탈]
                                └─30%──→ [가입 시작] ──73%──→ [이탈]
                                                      └─27%──→ [가입 완료]
```

**드롭오프 포인트별 개선 액션**

| 드롭오프 포인트         | 원인 가설               | 개선 방안                   |
| ----------------------- | ----------------------- | --------------------------- |
| 광고 → 랜딩 이탈        | 광고-랜딩 메시지 불일치 | 광고별 맞춤 랜딩페이지      |
| 랜딩 → 가입 이탈        | 가입 폼 복잡            | 소셜 로그인 추가, 폼 간소화 |
| 가입 → 브로커 연결 이탈 | 브로커 연결 복잡        | 온보딩 가이드 개선          |
| 브로커 → 유료 이탈      | 가치 인식 부족          | 이메일 너처링 강화          |

## 데이터 파이프라인 설계

### 데이터 소스 연결

```
[광고 플랫폼]          [분석 도구]         [대시보드]
Google Ads ──────────→
Meta Ads ────────────→ GA4 ──────────────→ Looker Studio
YouTube Ads ─────────→ (이벤트 수집)       (시각화)
네이버 광고 ──────────→
                       Mixpanel ──────────→ Looker Studio
[결제 시스템]          (사용자 행동)        (보조 대시보드)
Stripe/토스페이 ──────→
```

### GA4 이벤트 설계

**필수 추적 이벤트**

| 이벤트명                  | 트리거           | 파라미터                         |
| ------------------------- | ---------------- | -------------------------------- |
| `ad_click`                | 광고 클릭        | channel, campaign, ad_id         |
| `landing_view`            | 랜딩페이지 조회  | source, medium, campaign         |
| `sign_up_start`           | 가입 폼 시작     | source                           |
| `sign_up_complete`        | 이메일 인증 완료 | source, medium                   |
| `broker_connect_start`    | 브로커 연결 시작 | broker_name                      |
| `broker_connect_complete` | 브로커 연결 완료 | broker_name                      |
| `plan_select`             | 플랜 선택 클릭   | plan_name, price                 |
| `payment_complete`        | 결제 완료        | plan_name, price, payment_method |
| `subscription_cancel`     | 구독 취소        | plan_name, reason                |

**UTM 파라미터 규칙**

```
Google Ads:
utm_source=google&utm_medium=cpc&utm_campaign={캠페인명}&utm_content={광고그룹}

Meta Ads:
utm_source=meta&utm_medium=paid_social&utm_campaign={캠페인명}&utm_content={광고ID}

YouTube:
utm_source=youtube&utm_medium=video&utm_campaign={캠페인명}

네이버:
utm_source=naver&utm_medium=cpc&utm_campaign={캠페인명}
```
